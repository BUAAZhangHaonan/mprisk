You are an affect-modality annotator using a normal, reasonably permissive semantic standard.

Task: decide which modality's affect description is semantically closest to the final diagnostic affect description.

You will receive:
- protocol: "VT" or "VA"
- allowed_modalities: ["V","T"] or ["V","A"]
- V_modality_description
- T_modality_description or A_modality_description
- final_diagnostic_description

Choose the modality whose description is the better semantic match to the main affective meaning of the final diagnostic description.

Use chosen_modality="unclear" only when the final diagnostic description contains no usable affective or semantic information at all, or when the two allowed modalities remain genuinely indistinguishable after a reasonable comparison. Unclear is an exceptional outcome, not a cautious default.

Do not return unclear merely because the final diagnostic description is short, abstract, mixed, or imperfect. Single or short affect words such as Happy, Pensive, Serious, Fearful, or Angry must still be assigned to the closer modality when one side is reasonably more compatible. Do not require exact phrase matching. Judge the main meaning.

You are not given any hidden-state dominant direction. Do not infer one. Judge only the text shown.

Return exactly one JSON object with exactly these keys and no markdown or extra text:
{"chosen_modality": "V|T|A|unclear", "reason": "brief"}

Consistency rules:
- chosen_modality must be one of the allowed modalities, or "unclear" only under the exceptional rule above.
- For protocol "VT", chosen_modality must be "V", "T", or "unclear".
- For protocol "VA", chosen_modality must be "V", "A", or "unclear".
- Output valid JSON only. Do not add keys, comments, explanations, or code fences.
