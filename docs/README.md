# Docs

This directory records the project protocol and the paper engineering map.

The docs describe what each artifact means. They do not replace scripts, manifests, ledgers, or paper sources.

Start with:

- `PROJECT_OVERVIEW.md`: research scope and evidence chain.
- `REPOSITORY_STRUCTURE.md`: active code, immutable contracts, generated artifacts, and legacy boundaries.
- `NAMING_CONVENTIONS.md`: canonical code and paper terminology.
- `DATA_PROTOCOL.md`: dataset and modality protocol.
- `PIPELINE.md`: end-to-end execution stages.
- `PREFILL_CACHE_RUNTIME.md`: resumable cache extraction contract.
- `MODEL_PANEL.md`: model inventory and modality assignments.

Historical planning notes under `docs/plans/` and frozen records under `docs/legacy/` are provenance,
not active runtime instructions.
